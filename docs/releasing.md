# Releasing Codex Account Switcher

This project is a SwiftPM menubar app, so the release path is script-driven rather than Xcode-archive-driven.

## One-time setup

1. Generate Sparkle Ed25519 keys:
   ```bash
   ./scripts/generate_sparkle_keys.sh
   ```
2. Store the private key safely.
3. Copy the public key into your release environment as `SPARKLE_PUBLIC_ED_KEY`.
4. Use the public release-assets repo for distribution:
   - repo: `https://github.com/naveenshaji/codex-account-switcher-releases`
   - appcast URL: `https://raw.githubusercontent.com/naveenshaji/codex-account-switcher-releases/main/appcast.xml`
5. Create a notarytool keychain profile once on this Mac:
   ```bash
   xcrun notarytool store-credentials codex-account-switcher \
     --apple-id "you@example.com" \
     --team-id "TEAMID1234" \
     --password "app-specific-password"
   ```

## Build the signed app

```bash
SIGNING_IDENTITY="Developer ID Application: Your Name (TEAMID1234)" \
SPARKLE_PUBLIC_ED_KEY="<public-ed25519-key>" \
./scripts/build_release_app.sh \
  --version 0.1.0 \
  --build 1 \
  --appcast-url https://raw.githubusercontent.com/naveenshaji/codex-account-switcher-releases/main/appcast.xml
```

Output:
- `dist/release/Codex Account Switcher.app`

Notes:
- The build script creates a universal (`arm64 + x86_64`) binary.
- Sparkle helpers are embedded and re-signed with the host app.
- Host app signing uses `Packaging/Release.entitlements`, which includes Apple Events permission for Terminal automation.

## Notarize and staple

```bash
NOTARY_KEYCHAIN_PROFILE="codex-account-switcher" \
./scripts/notarize_release.sh
```

Output:
- stapled app bundle in `dist/release/`
- distributable zip in `dist/release/Codex Account Switcher-<version>.zip`

## Generate appcast

1. Copy the notarized zip into an appcast staging directory, for example `dist/appcast/`.
2. Optionally add release notes beside it using the same base filename:
   - `Codex.Account.Switcher-0.1.0.html`
   - `Codex.Account.Switcher-0.1.0.md`
3. Generate or update the appcast:
   ```bash
   ./scripts/generate_appcast.sh \
     --archives-dir dist/appcast \
     --download-url-prefix https://github.com/naveenshaji/codex-account-switcher-releases/releases/download/v0.1.0/ \
     --release-notes-url-prefix https://raw.githubusercontent.com/naveenshaji/codex-account-switcher-releases/main/
   ```

Output:
- `dist/appcast/appcast.xml`
- optional delta files generated by Sparkle

4. Publish the generated `appcast.xml` and release notes files to the root of the public release-assets repo and upload the notarized zip to the matching GitHub Release tag (for example `v0.1.0`).

## Verify before shipping

Run these checks on the stapled app:

```bash
codesign --verify --deep --strict --verbose=2 "dist/release/Codex Account Switcher.app"
spctl --assess --type execute --verbose=4 "dist/release/Codex Account Switcher.app"
xcrun stapler validate "dist/release/Codex Account Switcher.app"
```

## Required release environment

- `SIGNING_IDENTITY`: Developer ID Application certificate name
- `SPARKLE_PUBLIC_ED_KEY`: public Ed25519 key from Sparkle
- `NOTARY_KEYCHAIN_PROFILE`: preferred auth path for Apple notarization

Fallback notarization environment if not using a keychain profile:
- `APPLE_ID`
- `APPLE_TEAM_ID`
- `APPLE_APP_PASSWORD`
